<objective>
## Goal
Build the mechanic-facing UI: a setup/profile screen for mechanics to create their shop profile, a dashboard showing their active assignments with job completion, and role-aware routing from HomeScreen.

## Purpose
Mechanics need their own entry point when they open the app. Without this, a mechanic-role user lands on the car owner HomeScreen with no way to access their jobs.

## Output
4 new files (MechanicSetupScreen, MechanicSetupScreenModel, MechanicDashboardScreen, MechanicDashboardScreenModel) and 2 modified files (HomeScreen, AppModule).
</objective>

## AC-1: Mechanic setup screen lets mechanic create their profile
```gherkin
Given a mechanic user with no existing mechanic_profiles row
When they open MechanicSetupScreen
Then they see form fields for shop name (required), shop type (dropdown), bio, city, state, years experience, hourly rate
When they fill in shop name and tap Save
Then upsertMechanicProfile is called and on success they are navigated to MechanicDashboardScreen
```

## AC-2: Mechanic dashboard shows active assignments
```gherkin
Given a mechanic user with active mechanic_assignments rows
When MechanicDashboardScreen loads
Then it shows a list of their active assignments with vehicle info and assignment notes
When they tap "Complete" on an assignment
Then completeAssignment is called and the item is removed from the list
```

## AC-3: Mechanic sees their profile / can edit it from dashboard
```gherkin
Given a mechanic on the dashboard
When they tap "Edit Profile"
Then MechanicSetupScreen opens with their existing profile pre-filled
When they save changes
Then the profile is updated and they return to the dashboard
```

## AC-4: HomeScreen routes mechanic users to MechanicDashboardScreen
```gherkin
Given a user with isMechanic == true who has authenticated
When HomeScreen's LaunchedEffect fires
Then it pushes MechanicDashboardScreen (or MechanicSetupScreen if no profile) instead of staying on HomeScreen owner view
```
