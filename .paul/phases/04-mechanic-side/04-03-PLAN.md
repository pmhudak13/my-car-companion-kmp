<objective>
## Goal
Let admins see and verify pending mechanics from the admin panel. Adds a "Mechanics" tab to AdminScreen showing all mechanic profiles with their verification status, and Approve/Reject buttons for pending ones. Approving a mechanic sets verification_status to "verified" and adds/confirms the "mechanic" role in user_roles.

## Purpose
The mechanic side is only useful if mechanics can get verified. Without admin verification UI, the admin has no way to approve a mechanic — profiles just sit in "pending" forever.

## Output
2 modified UI files and 1 modified repository file (ProfileRepository, already modified in plan 01 — add mechanic admin methods).
</objective>

    - Update `mechanic_profiles` set `verification_status = "rejected"` where `user_id = userId`.
